# 学习与实现路线

## 1. 目标
- 用最稳的顺序完成 `tests/adapters.py` 里的 21 个函数。
- 每完成一小步就跑对应测试，避免一次性堆太多错误。
- 代码实现建议放在 `cs336_basics/` 自建模块中，`tests/adapters.py` 只做薄封装。

## 1.1 推荐代码组织
- 推荐新建模块：
  - `cs336_basics/model.py`
  - `cs336_basics/nn_utils.py`
  - `cs336_basics/optimizer.py`
  - `cs336_basics/data.py`
  - `cs336_basics/tokenizer.py`
  - `cs336_basics/serialization.py`
- 原则：
  - 复杂逻辑放在 `cs336_basics/`，便于复用和调试。
  - `tests/adapters.py` 只负责调用你自己的实现函数/类。

## 2. 总体节奏
1. 先做最小数学组件（最快出正反馈）。
2. 再做数据采样。
3. 再做模型底层算子。
4. 再拼装 Transformer Block / LM。
5. 再做优化器、学习率、checkpoint。
6. 最后做 tokenizer + train_bpe（最容易卡住，放后面）。

## 3. 分阶段执行清单

### 阶段 A：先拿到第一批通过
- 学习重点：
  - Tensor 基础操作（逐元素、广播、dim）
  - 激活/softmax/cross_entropy 的数值稳定性
- 实现函数：
  - `run_silu`
  - `run_softmax`
  - `run_cross_entropy`
  - `run_gradient_clipping`
- 测试命令：
```bash
uv run pytest tests/test_nn_utils.py -q
uv run pytest tests/test_model.py::test_silu_matches_pytorch -q
```

### 阶段 B：数据采样
- 学习重点：
  - 序列切片、随机采样、device 放置
- 实现函数：
  - `run_get_batch`
- 测试命令：
```bash
uv run pytest tests/test_data.py -q
```

### 阶段 C：模型底层组件
- 学习重点：
  - 线性层、Embedding、RMSNorm、RoPE、SDPA、SwiGLU
- 实现函数：
  - `run_linear`
  - `run_embedding`
  - `run_rmsnorm`
  - `run_scaled_dot_product_attention`
  - `run_rope`
  - `run_swiglu`
- 建议顺序：
  - `linear -> embedding -> rmsnorm -> silu/swiglu -> sdpa -> rope`
- 测试命令：
```bash
uv run pytest tests/test_model.py::test_linear -q
uv run pytest tests/test_model.py::test_embedding -q
uv run pytest tests/test_model.py::test_rmsnorm -q
uv run pytest tests/test_model.py::test_swiglu -q
uv run pytest tests/test_model.py::test_scaled_dot_product_attention -q
uv run pytest tests/test_model.py::test_rope -q
```

### 阶段 D：模型拼装
- 学习重点：
  - 多头注意力张量变形
  - pre-norm block 结构
  - 整体前向流
- 实现函数：
  - `run_multihead_self_attention`
  - `run_multihead_self_attention_with_rope`
  - `run_transformer_block`
  - `run_transformer_lm`
- 测试命令：
```bash
uv run pytest tests/test_model.py::test_multihead_self_attention -q
uv run pytest tests/test_model.py::test_multihead_self_attention_with_rope -q
uv run pytest tests/test_model.py::test_transformer_block -q
uv run pytest tests/test_model.py::test_transformer_lm -q
uv run pytest tests/test_model.py -q
```

### 阶段 E：训练工程组件
- 学习重点：
  - AdamW 参数更新逻辑
  - cosine warmup schedule
  - 模型与优化器状态序列化
- 实现函数：
  - `get_adamw_cls`
  - `run_get_lr_cosine_schedule`
  - `run_save_checkpoint`
  - `run_load_checkpoint`
- 测试命令：
```bash
uv run pytest tests/test_optimizer.py -q
uv run pytest tests/test_serialization.py -q
```

### 阶段 F：Tokenizer 与 BPE 训练
- 学习重点：
  - BPE merge 规则
  - special token（包括重叠 special token）
  - 速度优化（`train_bpe_speed`）
- 实现函数：
  - `get_tokenizer`
  - `run_train_bpe`
- 测试命令：
```bash
uv run pytest tests/test_tokenizer.py -q
uv run pytest tests/test_train_bpe.py -q
```

## 4. 每天怎么做（建议）
- 每次只攻克 1-2 个函数。
- 每做完一个函数就跑“最小相关测试”。
- 测试通过后再写笔记，记录：
  - 公式
  - 输入输出 shape
  - 你踩过的坑

## 5. 什么时候算“可以收工”
- 全量测试命令：
```bash
uv run pytest -q
```
- 目标结果（macOS 常见）：
  - 46 passed, 2 skipped
- 且 `tests/adapters.py` 中无 `NotImplementedError`。

## 6. 常见卡点（提前避坑）
- 只看一个大失败列表，不拆分最小测试。
- 张量维度写反（特别是 MHA 的 reshape/rearrange）。
- softmax/cross_entropy 没做数值稳定处理。
- tokenizer 一上来就全做，导致长时间卡住。
